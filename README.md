## ЁЯОп TL;DR

**тИВ-NO (del-NO)** is a drop-in augmentation that injects **learnable fractional derivative features** into any neural operator backbone. Both the derivative orders $\beta_k$ and feature scales $s_k$ are jointly optimized with network weights via a differentiable Gr├╝nwaldтАУLetnikov stencil. **The central finding:** the MSE-optimal derivative order $\beta^*$ is *strictly less* than the PDE order $m$, with the gap closed-form predicted by a spectral biasтАУvariance tradeoff (Theorem 5.6).

## Why тИВ-NO

- **ЁЯФМ Drop-in** тАФ wraps any backbone with no architectural change.
- **ЁЯУР Theory-grounded** тАФ closed-form $\beta^* < m$, validated on Burgers (smooth and near-shock).
- **ЁЯУК Consistent gains** тАФ 40/40 cells in the main table (5 PDEs ├Ч 4 backbones ├Ч 2 noise regimes); 120/120 under Gaussian + uniform stress noise; 14/14 for тИВ-CNO; 8/8 for тИВ-PINO.
- **ЁЯМА Differentiable & light** тАФ gradients flow through $\partial_\beta$; modest overhead and no extra hyperparameter search.

## The method, in one line

```python
# тИВ-NO augmentation тАФ wraps any backbone NO_theta
x_aug = torch.cat([x, u, *[s_k * D_beta_k(u) for k in range(K)]], dim=-1)
y     = NO_theta(x_aug)          # beta_k and s_k are learned end-to-end
```

Each `D_beta_k` is a Gr├╝nwaldтАУLetnikov fractional derivative with a *learnable* order $\beta_k$ and *learnable* scale $s_k$. The implementation is included **inline** in every notebook тАФ no heavy dependencies beyond what your backbone already needs.

## Data

The notebooks use the 1D Burgers datasets from [Li et al. 2020](https://github.com/neuraloperator/neuraloperator) тАФ ╬╜=0.1 (smooth) and ╬╜=0.001 (near-shock) тАФ placed in `data/`. The KdV set used in the paper is generated by direct numerical integration of $\partial_t u + u\,\partial_x u + \delta\,\partial_{xxx}u = 0$ with $\delta = 2.5\times10^{-3}$ from smooth random initial conditions (paper App. I). All third-party datasets are used under their respective licenses for academic research.

## Tips

- **GL stencil length:** `K_GL = 16` (default; `8` is acceptable, below 8 degrades fractional fidelity тАФ App. L.9).
- **╬▓ learning rate:** use `10├Ч` the backbone LR (╬▓ sits at the input level, so gradients attenuate through the full network).
- **Channel count:** `K=2` for 1D, `K=5` for 2D (4 directional + 1 mixed partial), `K=3` for KdV; initialize at $\beta = 1.0, 2.0$ (add $\beta_3 = 3.0$ for KdV).
- **Stability:** the GL stencil omits the $h^{-\beta}$ factor (absorbed by the learnable scales $s_k$); if losses go `nan` early or ╬▓ stalls, check the ╬▓ learning rate / gradient magnitudes.
- **Coarser grids amplify the gain:** тИВ-NO's relative advantage grows as the network's Nyquist headroom shrinks (App. L.4).
